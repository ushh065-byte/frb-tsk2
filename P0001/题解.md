## 贤者的试炼 - 题解

### 题意分析

给定 $n \in \{32, 64\}$ 和 $1 \leq k \leq 2^n - 1$，求 $2^n - k - 1$。

注意到，在计算机的补码表示中，对于 $n$ 位无符号整数 $k$，按位取反 `~k` 的结果恰好是：

$$\sim k = 2^n - k - 1$$

**原理**：
- $n$ 位无符号整数的每一位取反后，相当于用全 $1$（即 $2^n - 1$）减去原数
- 例如 8 位：`~00000101` = `11111010`
- 数学上：$(11111111)_2 - (00000101)_2 = (11111010)_2$
- 即：$2^8 - 1 - 5 = 255 - 5 = 250$

因此：
$$\sim k = (2^n - 1) - k = 2^n - k - 1$$

这正是我们要求的答案

### 标准程序

```c
#include<stdio.h>
int main(){
    unsigned int n,k32;
    unsigned long long k64;
    while(scanf("%d %llu",&n,&k64)!=EOF){
        if(n==64){
            printf("%llu\n",~k64);
        }
        else{
            k32=k64;  // 截断为32位
            printf("%u\n",~k32);
        }
    }
}
```

**关键点**：
1. 使用 `unsigned long long` 读入 $k$（因为32位和64位的 $k$ 都可能很大）
2. 对于 $n=32$，先将 $k$ 截断为 `unsigned int` 类型（32位），再取反
3. 对于 $n=64$，直接对 `unsigned long long` 取反
4. 输出时使用对应的格式符：`%u` 或 `%llu`

### 为什么要分类型处理？

如果不分类型，对于 $n=32$ 直接对 64 位的 $k64$ 取反：

```cpp
// 错误示例
printf("%llu\n", ~k64);  // 对64位取反
```

这会计算 $2^{64} - k - 1$ 而不是 $2^{32} - k - 1$

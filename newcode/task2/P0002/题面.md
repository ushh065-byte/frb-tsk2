题目：安全数据帧接收器（内存安全类）
题目难度：★★☆☆☆（中等）
题目背景
在无人机飞行控制系统中，飞控计算机需要通过串口实时接收来自地面站或传感器的数据帧。数据帧通常包含帧头、长度、有效数据和校验和，以确保传输的可靠性。然而，真实硬件环境中，电磁干扰或硬件故障可能导致数据帧错误（如长度字段异常、帧头丢失、校验失败），甚至可能因恶意攻击或软件缺陷导致接收缓冲区溢出，进而破坏关键内存区域，引发系统崩溃或失控。
本题目要求你在STM32嵌入式平台上实现一个安全可靠的UART数据帧接收模块，能够正确处理正常帧，并能优雅地处理异常情况，避免任何内存安全问题。
任务描述
给定以下数据帧格式：
字节偏移	内容	说明
0	帧头	固定为 0xAA
1	数据长度 L	1字节，表示有效数据的长度，
取值范围 1~16
2 ~ 2+L-1	有效数据	L 字节，需存入缓冲区
2+L	校验和	1字节，累加和校验（从帧头到数据最
后一字节的累加，取低8位）
你需要实现以下两个核心函数（框架代码已提供UART初始化及中断入口）：
c
/* 全局缓冲区（由平台提供，你可以在函数中使用） */
#define MAX_DATA_LEN 16
#define BUFFER_SIZE  (2 + MAX_DATA_LEN + 1)  // 帧头+长度+数据+校验和，最大20字节
uint8_t rx_buffer[BUFFER_SIZE];     // 用于暂存接收到的原始数据
uint8_t received_data[MAX_DATA_LEN]; // 最终提取的有效数据
volatile uint8_t data_ready;        // 当一帧完整接收且校验通过后置1

/* 你需要实现的函数 */
void UART_IRQHandler(void);          // 串口中断服务程序，每收到一个字节调用一次
uint8_t process_frame(uint8_t *raw, uint8_t len); // 解析并校验帧，成功返回1，失败返回0 
具体要求：
1.在中断服务程序 UART_IRQHandler 中，将接收到的字节依次存入 rx_buffer，并实现帧同步（即检测到帧头 0xAA 后开始计数，接收后续字节）。当接收完一帧（长度由帧中第二个字节决定）后，调用 process_frame 进行校验。
2.process_frame 应校验：
o数据长度 L 是否在 1~16 范围内；
o实际接收到的字节数是否等于 L+3（帧头+长度+数据+校验和）；
o校验和是否正确（累加和取低8位等于最后一个字节）。
3.若校验通过，将有效数据复制到 received_data，并将 data_ready 置1；若失败，则丢弃该帧，重新开始寻找帧头，且不能造成缓冲区溢出或内存越界。
4.禁止使用动态内存分配（如 malloc），所有缓冲区必须静态分配。
5.代码必须符合 MISRA C 基本规范（如禁止使用 sprintf、strcpy 等不安全函数，必须显式类型转换等）。
6.考虑中断与主循环的并发：data_ready 必须用 volatile 修饰，且主循环在读取后需及时清零。
输入格式
通过串口发送字节流（十六进制形式描述）。平台将模拟硬件发送一系列数据帧（可能包含错误帧），你的程序需要正确处理并输出结果。
输出格式
当主循环检测到 data_ready 为1时，通过串口打印一行：
text
OK: [数据内容]
其中 [数据内容] 为接收到的有效数据的十六进制表示，每个字节两位大写十六进制数，空格分隔。例如接收到数据 0x01 0x02 0x03，输出 OK: 01 02 03。
如果收到错误帧（校验失败等），程序应静默丢弃，不输出任何信息，并准备接收下一帧。
样例
样例1（正常帧）
输入（十六进制字节流）：
text
AA 03 11 22 33 69
解析：
帧头 AA
长度 03
数据 11 22 33
校验和 69 = (AA+03+11+22+33) & 0xFF
输出：
text
OK: 11 22 33
样例2（长度超限帧）
输入：
text
AA 11 01 02 03 ... (后续字节省略，长度11超出了允许的1~16)
要求：检测到长度字段 0x11（17）超出范围，丢弃该帧，继续等待下一个帧头。
样例3（校验错误帧）
输入：
text
AA 02 55 AA 00
计算校验和：AA+02+55+AA = 0x15B，取低8位 0x5B，但最后一个字节是 0x00，校验失败，丢弃。
样例4（连续多帧混合）
输入：
text
AA 01 55 AA   AA 02 66 77 44   AA 10 ... (错误帧) ...
程序应正确识别并输出正确帧，错误帧静默忽略。
提示（Tips）
在中断中处理数据时，务必考虑临界区保护（如关中断）以避免主循环同时访问 rx_buffer，但本题简化起见，假设主循环仅在 data_ready 置位后才访问 received_data，因此只需保证中断内对共享变量的访问是原子的（data_ready 用 volatile 即可）。
实现帧同步时，需要维护一个状态机：空闲、接收中、已完成。注意在接收过程中如果遇到另一个帧头（0xAA）应该如何处理？建议重新同步。
为了防止缓冲区溢出，必须检查当前接收位置不超过 BUFFER_SIZE-1，若超限则丢弃并复位状态。
校验和计算可使用循环累加，注意类型为 uint8_t 自动截断。
在真实硬件上运行时，由于串口接收可能产生背靠背中断，需确保中断服务程序足够快，避免丢失字节。
